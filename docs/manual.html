0개요 & 준비물

설치부터 정상 동작 판정, 그리고 초기 구축에서 마주친 두 겹의 소프트웨어 문제(칩 오인 · ESP32 I2C 드라이버 비호환)의 진단 과정과 증상별 해결까지 한 곳에 정리한 운용 문서입니다.

무엇을 만드나

MLX90641 적외선 어레이 센서(16×12, 192픽셀)가 읽은 온도 프레임을 Arduino Nano ESP32가 I2C로 읽어 보정 계산한 뒤 USB 시리얼로 PC에 전송하고, 브라우저의 Web Serial HTML 앱이 이를 실시간 열화상으로 렌더링합니다. 클라우드도 모델 학습도 없는, 센서 → 보정 → 스트리밍 → 시각화의 순수 측정·시각화 파이프라인입니다.

이 매뉴얼을 보는 순서

처음 구축이면 왼쪽 메뉴를 위에서부터 차례대로(설치 → 펌웨어 → 점검 → UI) 따라가면 동작합니다. 값이 이상하면 동작 점검(3)과 증상별 진단표(!)로, 원인 배경이 궁금하면 진단 타임라인(7)·라이브러리 이슈 분석(8)을 보세요. 코드 블록 오른쪽 위 📋 복사 버튼으로 내용을 그대로 복사할 수 있습니다.

각 부분의 역할

구성하는 일비유
MLX90641물체가 내는 적외선을 192개 픽셀로 감지, 원시 데이터와 공장 보정값(EEPROM) 제공눈(망막)
Nano ESP32I2C로 프레임을 읽어 보정 수학으로 온도(℃)로 변환, 시리얼로 스트리밍시신경·번역가
Web Serial 앱시리얼 줄을 파싱, 깨진 프레임은 폐기, 컬러맵을 입혀 열화상으로 렌더링화가·전광판
시리얼 프로토콜 — 프레임당 한 줄 보드는 부팅 시 INFO:CHIP=MLX90641,COLS=16,ROWS=12 한 줄로 칩·해상도를 알리고, 이후 프레임마다 아래 형식의 한 줄을 115200bps로 보냅니다. 
INFO:CHIP=MLX90641,COLS=16,ROWS=12    # 부팅 1회 — 해상도 자동 통지
START:t0,t1,t2, ... ,t191:END          # 프레임당 한 줄 (192개 온도값, ℃)
예) START:25.31,25.28,24.97, ... ,26.02:END
UI 앱은 START:로 시작해 :END로 끝나고 값이 정확히 192개인 줄만 화면에 반영합니다(유효성 게이트). 전송 중 깨진 줄은 조용히 버려집니다. 상세 형식과 오류 코드는 패널 6 참조.
💡 왜 이렇게 단순한가프로토콜이 사람이 읽을 수 있는 텍스트 한 줄이라, IDE 시리얼 모니터만 열어도 UI 없이 동작을 검증할 수 있습니다(패널 3의 점검). 문제가 생기면 "보드까지는 정상인가 / UI부터 문제인가"를 바로 갈라낼 수 있습니다.

1라이브러리 설치

동봉된 ZIP 하나만 설치하면 됩니다. driver.cpp를 손으로 고치는 과거 방식은 더 이상 필요 없습니다.

설치 절차 (Arduino IDE)

  1. Arduino IDE 메뉴에서 스케치 → 라이브러리 포함하기 → .ZIP 라이브러리 추가… 선택
  2. 저장소의 lib/MLX9064x_Universal.zip 선택 → 설치 완료
  3. 설치 확인: 파일 → 예제 → MLX9064x_Universal 에 01_Diagnostic, 02_SerialStream_UI, 03_BasicReadings 세 예제가 보이면 정상
왜 ZIP 한 번이면 끝인가이 ZIP에는 ESP32에서 깨지던 I2C 드라이버를 재작성한 패치본과 칩 자동판별(EEPROM 0x240A bit6)이 이미 들어 있습니다. 따라서 driver.cpp를 손으로 고칠 필요가 없고, 라이브러리를 재설치해도 패치가 유실되지 않습니다. 보정 수학(Melexis 공식)은 원본 그대로라 정확도는 보장됩니다.
💡 이미 공식 라이브러리가 설치돼 있다면공식 Seeed 라이브러리(Seeed_Arduino_MLX9064x)가 설치된 상태라면 헤더명이 겹쳐 IDE가 엉뚱한 쪽을 컴파일할 수 있습니다. 기존 것을 제거(또는 폴더 이동)한 뒤 ZIP을 설치하세요. 공식판을 그대로 쓰면 ESP32에서 프레임이 동결되는 증상이 납니다(증상별 진단표 참조).

2펌웨어 업로드

스트리밍 펌웨어를 올립니다. 스케치는 래퍼 헤더 하나만 include하면 되고 칩은 자동 판별됩니다.

업로드 절차

  1. Grove 케이블로 센서를 I2C 포트(SDA=A4, SCL=A5)에 연결하고 전원 LED를 확인합니다.
  2. firmware/mlx90641_serial.ino를 엽니다 (라이브러리 예제 02_SerialStream_UI와 동일 — 파일 → 예제 → MLX9064x_Universal에서도 열 수 있음).
  3. 보드 Arduino Nano ESP32 선택, 포트 선택 후 업로드.
  4. 스케치는 래퍼 헤더 MLX9064x_Universal.h 하나만 include 하면 되고, 칩(90641/90640)은 자동 판별되므로 직접 고를 필요가 없습니다.

핵심 동작 (스케치 요약)

firmware/mlx90641_serial.ino 
#include <MLX9064x_Universal.h>
MLX9064xUniversal cam;
float frame[MLX9064X_MAX_PIXELS];   // 어느 칩이든 안전한 크기(768)

void setup() {
  Serial.begin(115200);
  cam.begin();                      // Wire 초기화 + 자동판별 + 보정 로드
  cam.setRefreshRate(MLX9064X_RATE_8HZ);
  // 부팅 1회: INFO:CHIP=MLX90641,COLS=16,ROWS=12
}

void loop() {
  if (cam.readFrame(frame) != 0) return;          // 실패 시 ERR_FRAME
  Serial.print("START:");
  for (uint16_t i=0; i<cam.pixelCount(); i++) {   // 192 또는 768 자동
    Serial.print(frame[i], 2);
    if (i < cam.pixelCount()-1) Serial.print(",");
  }
  Serial.println(":END");
}
✅ 업로드 후 1차 확인시리얼 모니터(115200)에서 부팅 시 INFO:CHIP=MLX90641,COLS=16,ROWS=12 한 줄 → 이어서 START:25.31,…:END 줄이 끊김 없이 흐르고 값이 실온(20~30℃)이면 정상입니다. 센서 앞에 손을 대면 해당 구간 값이 33℃ 안팎으로 올라야 합니다.
⚠️ 값이 이상하면전 픽셀 NaN, Vdd=inf, 수천 ℃ 같은 가짜 값, 프레임마다 동일한 동결 값이 보이면 하드웨어가 아니라 라이브러리 문제일 가능성이 높습니다. 다음 패널(동작 점검)과 증상별 진단표(!)를 따라가세요.

3동작 점검 (진단)

스트리밍 펌웨어를 올리기 전(또는 값이 의심될 때) 다음 두 가지로 단계적으로 점검합니다.

A. 라이브러리 무관 raw 점검 — i2c_diagnostic.ino

Wire.h만 사용하는 30초 진단 스케치(firmware/i2c_diagnostic.ino)입니다. 시리얼 모니터(115200)에서:

출력의미
ACK: OK0x33 응답 — 배선·전원 정상
0x240A=0x.... -> MLX90641(16x12)EEPROM bit6 기반 칩 판별 결과
재독취 일치(통신 OK)같은 레지스터 2회 값 일치 — 신호 무결성

B. 라이브러리 예제 — 01_Diagnostic

자동판별·보정까지 포함해 프레임 품질을 봅니다. 한 줄 출력 예:

Vdd=3.18V Ta= 28.4C | center=  24.9 min=  23.1 max=  25.7 NaN=0

센서에 손을 대면 center·max가 33~36℃로 반응해야 정상입니다. 오류 시 readFrame ERR <code> / begin() FAILED, lastError=<code> 형태로 코드를 출력합니다(코드표는 패널 6 참조).

💡 진단 순서 요약① i2c_diagnostic.ino로 배선(ACK)·칩(bit6)·통신(재독취) 3종 점검 → ② 01_Diagnostic 또는 시리얼 모니터에서 START:...:END와 실온값 확인 → ③ 그다음에 UI를 붙입니다. 층을 나눠 확인하면 원인 구간이 바로 좁혀집니다.

4UI 실행 (Web Serial)

브라우저에서 HTML 파일 하나만 열면 됩니다. 서버도 설치도 필요 없습니다.

실행 순서

  1. IDE 시리얼 모니터를 닫습니다. 열려 있으면 포트를 점유해 UI가 연결되지 않습니다 (가장 흔한 실패 원인).
  2. ui/mlx90641_thermal_ui.html을 Chrome/Edge에서 엽니다 (더블클릭, 서버 불필요). 또는 GitHub Pages 버전을 바로 열어도 됩니다.
  3. 통신 속도 115200 확인 → 장치 연결 클릭 → 팝업에서 Nano ESP32 포트 선택.
  4. 센서 앞에 손을 비추면 핫스팟 십자선(H)이 손을 따라가고 최고 온도가 33~36℃를 표시해야 합니다.
✅ 정상 신호부팅 직후 INFO:CHIP=... 한 줄, 이어서 START:...:END 프레임이 끊김 없이 흐르고, 배경 픽셀이 실온±2℃에서 미세하게 요동치면 정상입니다.
⚠️ 가장 흔한 연결 실패 원인 — 포트 점유Arduino IDE의 시리얼 모니터가 열려 있으면 포트를 점유해서 브라우저가 같은 포트를 열 수 없습니다. UI에서 포트가 안 보이거나 연결이 거부되면 제일 먼저 시리얼 모니터(및 시리얼 플로터)를 닫으세요. Safari·Firefox는 Web Serial을 지원하지 않아 동작하지 않습니다.

-13보정 파라미터 추출 실패칩-API 불일치 / EEPROM 손상
-20프레임 I2C 실패케이블 길이·노이즈, 클럭 하향

7진단 타임라인 — 무엇이 문제였나

초기 구축에서 화면에는 H:7,844℃ 같은 말도 안 되는 핫스팟과 다수의 NaN·Vdd=inf가 떴습니다. 원인은 하드웨어가 아니라 두 겹의 소프트웨어 문제였고, 다음 순서로 좁혀 나갔습니다.

  • 증상 포착2026-06

    전 픽셀 NaN·Vdd=inf, 가끔 H:7844℃의 가짜 핫스팟. 배선을 여러 번 점검해도 동일 → 하드웨어 단독 원인 가능성 배제.

  • ① 칩 오인 발견근본원인 1

    모듈을 MLX90640(32×24)으로 가정하고 90640 API를 쓰고 있었으나, EEPROM 0x240A bit6을 직접 읽어 보니 실제 칩은 MLX90641(16×12)이었다. 칩-API 불일치는 보정 추출을 망가뜨려 Vdd=inf·전픽셀 NaN을 만든다. → MLX90641 API로 전환.

  • ② ESP32 I2C 드라이버 비호환근본원인 2

    API를 맞췄는데도 프레임이 동결(매 프레임 동일한 쓰레기 값)됐다. 공식/Seeed 드라이버의 I2C 읽기가 ESP32 Wire에서 NACK인데도 성공으로 보고하고 requestFrom 수신량을 검증하지 않아, 부분 수신·이전 데이터 잔류가 발생. → I2CRead를 로컬 패치(실패를 -1로 반환 + 수신량 엄격 검증 + 16워드 청크).

  • 재발 방지 + UI 보강대책

    두 수정을 한 라이브러리로 묶고(MLX9064x_Universal) 칩 자동판별을 더해 칩 오인을 원천 차단. UI에는 유효성 게이트(깨진 프레임 폐기)와 컬러맵·보간·시뮬레이터를 구성.

💡 교훈열화상 값이 물리적으로 불가능하면(수천 ℃, NaN) 배선보다 칩 판별과 라이브러리부터 의심하세요.

9보정 · 교육 활용

숫자가 어떻게 온도가 되는지 — 방사율·교정의 원리와 수업 활용 포인트를 정리합니다.

기본 보정값

  • 방사율(emissivity) 0.95 기본. 피부는 0.97~0.98, 금속면은 별도 보정 필요.
  • TA_SHIFT 8 기본. 정밀 측정 시 기준 온도계로 1점 교정.
  • API: cam.setEmissivity(float) · cam.setTaShift(float) · cam.getTa() / cam.getVdd()
💡 정밀 측정 — 1점 교정정밀하게 재야 한다면 기준 온도계(예: 접촉식)로 한 지점의 실제 온도를 재고, 열화상 표시값과의 차이만큼 오프셋을 맞추는 1점 교정을 하세요.

칩 판별 — 90641인가 90640인가

MLX90641(16×12)과 MLX90640(32×24)은 같은 I2C 주소(0x33)를 쓰는 형제 칩이라 겉으로 구분이 어렵습니다. 판별 기준은 EEPROM 0x240A bit6이며, 동봉 라이브러리가 자동으로 판별합니다(상세는 패널 8). 직접 확인하려면 i2c_diagnostic.ino를 쓰세요.

교육 포인트 — 흑체복사·적외선 단원과 연결

  • 모든 물체는 온도에 따라 적외선을 방출한다(흑체복사) — 센서는 이 복사를 192개 픽셀로 잰다.
  • UI에서 보간 OFF + 수동 스케일로 두면 손·금속·유리의 온도가 다르게 보이는 이유(방사율 차이)를 흑체복사·적외선·방사율 단원과 직접 연결해 토론할 수 있다.
  • 같은 실제 온도라도 방사율이 낮은 금속은 더 차갑게 측정되는 현상을 직접 확인시키는 활동이 가능하다 — "카메라가 온도를 찍는다"가 아니라 "복사 에너지를 재서 온도로 계산한다"는 점을 체감시킨다.

끝정리·보관

데모를 마치고 장비를 정리하는 순서입니다. 특별한 초기화 절차는 필요 없습니다.

연결 해제 순서

  1. UI 연결 해제 — 브라우저 앱에서 연결 끊기(또는 탭 닫기). 포트가 정상적으로 반납됩니다.
  2. USB 분리 — Nano ESP32의 USB-C 케이블을 뽑습니다. 펌웨어는 보드에 남아 있어 다음에 꽂으면 바로 스트리밍을 재개합니다.
  3. 센서 분리·보관 — Grove 케이블을 뽑고, 센서는 렌즈(적외선 창)에 지문·먼지가 닿지 않게 보관합니다.
